HTTP

如何理解HTTP是无状态的

无状态：可以理解为当浏览器向服务器发送请求，服务器响应了；浏览器又向相同服务器发起请求，服务器也会响应，但是服务器不会知道你是刚才那个浏览器

如何保证HTTP的状态

可以利用Cookie 和 Session

Cookie 和 Session 都是用来跟踪浏览器用户身份的会话方式

HTTP/1.0，1.1，2.0的区别

从网络IO上看：短连接（可以强制开启长连接，设置Connection: keep-alive） => 长连接 => 多路复用

HTTP1.1特点

  • 默认使用持久连接，即TCP默认不关闭，原理主要利用TCP的保活机制
  • 使用管道化技术pipeLining：可以在等待上一个请求响应的同时，发送下一个请求，其实就是把多个HTTP请求放到一个TCP连接中——发送；不过服务端还是按照发送请求的顺序处理请求，客户端还是按照发送请求的顺序来接受响应

HTTP2.0特点

  • 头部压缩，HTTP协议不带有状态，每次请求都必须附上所有信息。Http/2.0引入了头信息压缩机制，使用gzip或compress压缩后再发送
  • 二进制协议，1.1版本的头信息是文本（ASCII编码），数据体可以是文本或者二进制；2.0中，头信息和数据体都是二进制
  • 多路复用，在一个连接里，客户端和浏览器都可以同时发送多个请求或回应，而且不用按照顺序一一对应
  • 服务端推送，允许服务器未经请求，主动向客户端发送资源
